PROTOCOLLO COMUNICAZIONE CLIENT SERVER
0 - carettere terminatore conversazione
1 - creazione nuovo dominio
2 - visualizza prenotazione
3 - rinnovo prenotazione scaduta
4 - visualizza domini
5 - controllo disponibilità dominio
6 - visualizza ordini

--------------
status = "attivo"
	significa che il dominio è in funzione

status = "rinnovare"
	 significa che il dominio si trova in stallo per il rinnovo (bottone rosso)

status = "scaduto"
	significa che si ha scelto di non rinnovare il dominio

------------

Scaletta
1. divisione form registrazione con pagamento
2. quando un utente decide di registrare un nuovo dominio c'è un controllo al singolo input per verificare se è disponibile
3. quando conferma e va al pagamento, il quel momento il dominio è bloccato temporaneamente a livello server
4a. se il pagamento va a buon fine registro il dominio nel db e lo tolgo dalla struttura dati server per il bloccaggio
4b. se il pagamento non va a buon fine lo tolgo dalla struttura dati server per il bloccaggio
